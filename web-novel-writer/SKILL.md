# 网文创作工作流

此技能提供从零开始创作长篇网文的完整流程：设定搭建 → 大纲规划 → 逐章写作 → 状态追踪。

## 前置条件

开始之前，确保故事配置文件已存在且有内容：

1. 读取 `references/story-config.md` 检查填充状态
2. 读取 `references/writing-standards.md` 获取文风和格式规范
3. 如果 story-config 为空或大部分未填充，进入「设定阶段」
4. 如果已填充核心内容，直接进入「创作阶段」

## 阶段一：设定搭建

当故事配置为空或用户要求完善设定时执行。

### 1.1 引导填充

按以下优先级逐步引导用户填充 story-config.md，每次聚焦 1-2 个模块：

1. **作品基础信息** — 题材、基调、字数目标（必须最先确定）
2. **世界观 + 力量体系** — 故事的基础规则
3. **主角 + 核心配角** — 人物是故事的灵魂
4. **故事主线 + 三幕结构** — 剧情骨架
5. **分卷规划 + 支线** — 细化到可执行粒度

### 1.2 设定完善

当用户给出初步设定后，主动：
- **追问细节**：主角的性格缺陷是什么？世界观的铁律有哪些？
- **发现漏洞**：力量体系是否平衡？人物动机是否合理？
- **建议优化**：开篇钩子够不够强？爽点密度是否足够？
- **提供选项**：不确定时给 2-3 个方案供选择，而非泛泛提问

### 1.3 存入配置

每次讨论确定的设定，直接写入 `references/story-config.md` 对应位置。不等待、不拖延。

---

## 阶段二：大纲规划

设定完成后，生成分章节大纲。

### 2.1 生成章纲

基于 story-config 中的主线与分卷规划，生成详细的章节大纲：

```
第X章：章节标题
- 核心事件：（本章发生什么）
- 视角：（谁的角度）
- 信息量：（读者获得什么新信息）
- 情绪走向：（开头→结尾的情绪变化）
- 钩子：（结尾悬念）
- 字数预估：xxx
```

章纲生成后展示给作者审核，修改确认后再开始正文写作。

### 2.2 批量规划

一次性生成 10-30 章的章节大纲，方便作者把控整体节奏。

---

## 阶段三：逐章写作

### 3.1 写前准备

每一章开始前：

1. 读取 `references/story-config.md` 确认当前设定
2. 读取当前章节大纲
3. 回顾上一章结尾（衔接）
4. 确认本章要推进的主线/支线/伏笔

### 3.2 正文输出

按照 `references/writing-standards.md` 的风格和格式规范写作。使用 `assets/chapter-template.md` 的格式输出。

写完后自动在章节末尾附上创作备注：

```
【创作备注】
- 本章字数：xxxx
- 新建伏笔：
- 回收伏笔：
- 人物状态变化：
- 下章预告：
```

### 3.3 质量自检

每章写完后，自查：
- 爽点/钩子是否存在且有效？
- 人物性格是否前后一致？
- 对话是否像"人说的话"？
- 是否有无效灌水段落？
- 章末钩子是否足够吸引人？

---

## 阶段四：状态追踪

### 4.1 章节目录维护

在 story-config.md 末尾或独立文件中维护章节目录：

```
## 章节目录
| 章节 | 标题 | 字数 | 核心事件 | 状态 |
|------|------|------|----------|------|
| 1 | xxx | 3200 | xxx | ✅ |
| 2 | xxx | 2800 | xxx | ✅ |
| 3 | xxx | - | - | 📝 |
```

### 4.2 伏笔追踪

每章写完后立即更新 story-config.md 中的伏笔表格。标记新埋的伏笔、回收的伏笔。

### 4.3 人物状态追踪

重要人物的状态变化（等级提升、关系变化、获得新能力/物品等）在章节备注中记录，必要时同步到人物设定中。

---

## 交互原则

1. **不代替作者做关键决策**：情节走向、人物命运等重要决定交给作者
2. **提供备选方案**：当作者犹豫时，给 2-3 个明确方案 + 各方案的利弊
3. **接受推翻**：作者不满意随时推翻重来，不辩解
4. **保持一致性**：严格遵循已确定的设定，发现矛盾主动提醒
5. **适时提问**：设定模糊时及时追问，不自行脑补关键信息
